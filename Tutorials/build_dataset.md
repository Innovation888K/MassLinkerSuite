## Practical usage examples

This section provides practical examples for constructing MassLinker datasets from tokenized LCвЂ“MS files.

The examples below follow the current implementation of the Python dataset module.

---

## Example 1: Build a dataset from enhanced MassLinker tokens

If the MassLinker tokens were generated from augmented LCвЂ“MS files, use:

```python
from data import ExcelDataset

dataset = ExcelDataset(
    r"F:\MassLinkercodes_enhanced",
    r"target.xlsx",
    mode="enhance"
)

dataset.save_dataset(
    r"enhanced.joblib"
)
```

In this example:

| Argument | Description |
|---|---|
| `r"F:\MassLinkercodes_enhanced"` | Directory containing MassLinker token folders |
| `r"F:\targets\target.xlsx"` | Target file containing sample labels |
| `mode="enhance"` | Indicates that the sample folders contain augmentation prefixes |
| `8764_reduced_enhanced.joblib` | Saved processed dataset |

The input token directory should follow this structure:

```text
F:\MassLinkercodes
в”њв”Ђв”Ђ 1-sample_01\
в”‚   в””в”Ђв”Ђ 1-sample_01.xlsx
в”њв”Ђв”Ђ 2-sample_01\
в”‚   в””в”Ђв”Ђ 2-sample_01.xlsx
в”њв”Ђв”Ђ 1-sample_02\
в”‚   в””в”Ђв”Ђ 1-sample_02.xlsx
в””в”Ђв”Ђ 2-sample_02\
    в””в”Ђв”Ђ 2-sample_02.xlsx
```

When `mode="enhance"` is used, the prefix before the first hyphen is removed before matching sample names in the target file.

For example:

```text
1-sample_01 в†’ sample_01
2-sample_01 в†’ sample_01
```

Therefore, the target file should contain the original sample names rather than the augmented names.

Example target file:

| filename | label | is_positive |
|---|---|---:|
| sample_01 | disease_A | 1 |
| sample_02 | control | 0 |

## Example 2: Build a dataset from original MassLinker tokens

If the MassLinker tokens were generated from original non-augmented LCвЂ“MS files, use `mode="origin"`.

```python
from data import ExcelDataset

dataset = ExcelDataset(
    r"F:\MassLinkercodes",
    r"target.xlsx",
    mode="origin"
)

dataset.save_dataset(
    r"F:\dataset.joblib"
)
```

The input directory should follow this structure:

```text
MassLinkercodes\
в”њв”Ђв”Ђ sample_01\
в”‚   в””в”Ђв”Ђ sample_01.xlsx
в”њв”Ђв”Ђ sample_02\
в”‚   в””в”Ђв”Ђ sample_02.xlsx
в””в”Ђв”Ђ sample_03\
    в””в”Ђв”Ђ sample_03.xlsx
```

In `mode="origin"`, the folder name is directly matched to the `filename` column in `target.xlsx`.

## Example 3: Merge multiple processed datasets

For multi-cohort analysis, each cohort should first be processed and saved as an individual `.joblib` file.

Example processed dataset directory:

```text
F:\MassLinkercodes\multicohorts\
в”њв”Ђв”Ђ cohort_A_enhanced.joblib
в”њв”Ђв”Ђ cohort_B_enhanced.joblib
в””в”Ђв”Ђ cohort_C_enhanced.joblib
```

Then use `concat_dataset()` to merge them:

```python
from data import concat_dataset
import joblib

total_dataset = concat_dataset(
    r"F:\MassLinkercodes\multicohorts",
    r"init_polarity.xlsx"
)

joblib.dump(
    total_dataset,
    filename=r"merged_dataset_enhanced_total.joblib"
)
```

The `init_polarity.xlsx` file should contain:

| dataset | init_polarity |
|---|---|
| cohort_A_enhanced | high |
| cohort_B_enhanced | low |
| cohort_C_enhanced | high |

Important notes:

1. The `dataset` column must match the `.joblib` filename without extension.
2. Only datasets listed in `init_polarity.xlsx` will be merged.
3. If `init_polarity` is exactly `low`, the RBF center channel is reversed before concatenation.
4. Datasets marked as `high` are used directly.

For example:

```text
8764_reduced_enhanced.joblib в†’ 8764_reduced_enhanced
```

## Example 4: Load a merged dataset

A saved merged dataset can be loaded with `joblib`.

```python
import joblib

dataset = joblib.load(
    r"merged_dataset_enhanced_total.joblib"
)
```

If class IDs are needed, run:

```python
dataset.gen_classes()
```

The loaded dataset contains:

| Attribute | Description |
|---|---|
| `dataset.samples` | MassLinker token tensors |
| `dataset.label` | Sample labels |
| `dataset.name` | Sample names |
| `dataset.is_positive` | Positive/negative indicators |
| `dataset.classes` | Generated class IDs after calling `gen_classes()` |

## Example 5: Visualize MassLinker datasets with low-dimensional plots

The processed MassLinker tokens can be flattened and visualized using low-dimensional projection methods.

```python
import joblib
import utils

dataset = joblib.load(
    r"merged_dataset_enhanced_total.joblib"
)

dataset.gen_classes()

plotdata = [
    i.reshape(1, i.shape[0] * i.shape[1] * i.shape[2])[0].numpy()
    for i in dataset.samples
]

group = dataset.classes

utils.low_dim_plots(
    plotdata,
    group,
    dim=2
)
```

In this example:

| Variable | Description |
|---|---|
| `plotdata` | Flattened MassLinker token vectors |
| `group` | Class labels generated by `dataset.gen_classes()` |
| `dim=2` | Projects data into two dimensions |

This visualization can be used to inspect whether MassLinker tokens separate disease groups, control groups, or different cohorts.
