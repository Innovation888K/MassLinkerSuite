# Small-sample statistical comparison using MassLinker tokens

This tutorial describes how to compare two groups of samples using one MassLinker `ExcelDataset`-like object.

This workflow is designed for small-sample exploratory analysis. Instead of training a supervised prediction model, it directly compares MassLinker token-derived RBF curves between two groups, identifies differential metabolite tokens, visualizes differential peaks, and performs KEGG pathway enrichment analysis.

---

## Overview

The workflow uses one processed MassLinker dataset object as the core input.

The dataset object can be:

1. Constructed directly with `ExcelDataset`;
2. Loaded from a saved `.joblib` file;
3. Generated by merging multiple cohorts with `concat_dataset`.

The analysis includes:

1. Selecting samples from the dataset.
2. Extracting group labels from `dataset.is_positive`.
3. Extracting token parameters from `dataset.samples`.
4. Calculating JS divergence and Wasserstein distance using `metabo_dis()`.
5. Performing two-group statistical comparison using `get_diff()`.
6. Plotting top differential m/z or metabolite features.
7. Plotting reconstructed RBF curves for differential peaks.
8. Performing KEGG pathway enrichment analysis.
9. Visualizing enriched KEGG pathways.
10. Visualizing samples using significant distance features.

## Required input

The essential input is one MassLinker dataset object.

This object should be compatible with the current `ExcelDataset` structure and contain:

| Attribute | Description |
|---|---|
| `dataset.samples` | MassLinker token tensors for all samples |
| `dataset.is_positive` | Binary group indicator for each sample |
| `len(dataset)` | Number of samples |

For example, the dataset can be loaded from:

```python
import joblib

dataset = joblib.load("data/processed_dataset.joblib")
```

The `.joblib` file should contain an `ExcelDataset`-type object or a merged dataset object generated by MassLinker.

---

## Optional cached files

The distance calculation step can be time-consuming. Therefore, JS divergence and Wasserstein distance results can be saved and reused.

These files are optional cached results:

| File | Description |
|---|---|
| `cache/JS_distance.joblib` | Cached JS divergence result |
| `cache/Wasserstein_distance.joblib` | Cached Wasserstein distance result |

If these files do not exist, calculate them from the dataset:

```python
JS, was = metabo_dis(params)

joblib.dump(JS, filename="cache/JS_distance.joblib")
joblib.dump(was, filename="cache/Wasserstein_distance.joblib")
```

If they already exist, load them directly:

```python
JS = joblib.load("cache/JS_distance.joblib")
was = joblib.load("cache/Wasserstein_distance.joblib")
```

Both approaches use the same `params` extracted from the dataset.

---

## Required annotation file

This workflow also uses an annotation file:

```text
metadata/pathway_compound_detail.csv
```

This file provides compound names, m/z values, and KEGG pathway information.

It should contain at least the following columns:

| Column | Description |
|---|---|
| `compound_names` | Compound or metabolite names corresponding to MassLinker tokens |
| `mz` | m/z values |
| `kegg_id` | KEGG pathway ID |
| `pathway_name` | KEGG pathway name |

The row order of `pathway_compound_detail.csv` should match the token order in `dataset.samples`.

Example:

```text
metadata/
└── pathway_compound_detail.csv
```

## Complete example code

```python
from utils import (
    metabo_dis,
    get_diff,
    kegg_enrichment,
    plot_enrichment,
    plot_peak_comp,
    plot_2d,
    plot_p_value
)

import os
import joblib
import numpy as np
import pandas as pd


# Define project directories
data_dir = "data"
metadata_dir = "metadata"
cache_dir = "cache"
results_dir = "results"

peak_plot_dir = os.path.join(results_dir, "differential_peaks")
distance_plot_dir = os.path.join(results_dir, "distance_visualization")


# Create output directories if they do not exist
os.makedirs(cache_dir, exist_ok=True)
os.makedirs(results_dir, exist_ok=True)
os.makedirs(peak_plot_dir, exist_ok=True)
os.makedirs(distance_plot_dir, exist_ok=True)


# Define input and output files
dataset_path = os.path.join(data_dir, "processed_dataset.joblib")
annotation_path = os.path.join(metadata_dir, "pathway_compound_detail.csv")

JS_cache_path = os.path.join(cache_dir, "JS_distance.joblib")
Wasserstein_cache_path = os.path.join(cache_dir, "Wasserstein_distance.joblib")


# Load a processed MassLinker dataset.
# The loaded object should be an ExcelDataset-like object.
dataset = joblib.load(dataset_path)


# Select samples for analysis.
# Here, all samples in the dataset are selected.
sele_sample = [
    i
    for i in range(len(dataset))
]

# For testing or debugging, selected sample indices can be used instead.
# Example:
# sele_sample = [0, 1]


# Extract group labels.
# In this example, dataset.is_positive is used as the binary group indicator.
group = [
    dataset.is_positive[i].numpy().item()
    for i in sele_sample
]


# Extract MassLinker token parameters.
params = [
    dataset.samples[i]
    for i in sele_sample
]


# Calculate or load JS divergence and Wasserstein distance.
# This step can be time-consuming, so cached results are reused if available.
if os.path.exists(JS_cache_path) and os.path.exists(Wasserstein_cache_path):
    JS = joblib.load(JS_cache_path)
    was = joblib.load(Wasserstein_cache_path)
else:
    JS, was = metabo_dis(params)
    joblib.dump(JS, filename=JS_cache_path)
    joblib.dump(was, filename=Wasserstein_cache_path)


# Convert distance results to arrays.
was_arr = np.array(was)[0]
JS_arr = np.array(JS)[0]


# Perform two-group statistical comparison.
p_values, was_diff = get_diff(
    was_arr,
    group
)


# Load compound, m/z, and pathway annotation information.
compound_df = pd.read_csv(annotation_path)

met_name = compound_df["compound_names"].tolist()
mzs = compound_df["mz"].tolist()


# Plot top differential m/z features ranked by p-value.
plot_p_value(
    p_values,
    met_name,
    mzs=mzs,
    n=20,
    save_path=os.path.join(results_dir, "p_value_rank.pdf")
)


# Plot reconstructed RBF curves for top differential peaks.
plot_peak_comp(
    p_values,
    group,
    params,
    met_name,
    mzs,
    save_path=peak_plot_dir,
    top_n=2000
)


# Load pathway annotation table for KEGG enrichment.
pathway_df_full = pd.read_csv(
    annotation_path,
    index_col=0
)


# Perform KEGG enrichment analysis.
kegg_enriched = kegg_enrichment(
    pathway_df_full,
    p_values,
    was_diff
)


# Plot KEGG enrichment result.
plot_enrichment(
    kegg_enriched.copy(),
    save_dir=results_dir,
    save_name="KEGG_enrichment.pdf"
)


# Select significant distance features for 2D visualization.
was_selected = [
    []
    for i in range(len(was[0]))
]

JS_selected = [
    []
    for i in range(len(JS[0]))
]

for i in range(len(p_values)):
    if p_values[i] <= 0.01:
        for j in range(len(was[0])):
            was_selected[j].append(was[0][j][i])
            JS_selected[j].append(JS[0][j][i])


# Visualize samples using significant JS and Wasserstein distance features.
plot_2d(
    [JS_selected],
    [was_selected],
    group,
    save_path=distance_plot_dir
)
```

import joblib

## Step-by-step explanation

### Step 1: Select samples

```python
sele_sample = [
    i
    for i in range(len(dataset))
]
```

This selects all samples in the dataset.

For a small test run, selected indices can be used:

```python
sele_sample = [0, 1]
```

This is useful for debugging the workflow before running the full dataset.

---

### Step 2: Extract group labels

```python
group = [
    dataset.is_positive[i].numpy().item()
    for i in sele_sample
]
```

The group labels are extracted from `dataset.is_positive`.

Typical values are:

| Value | Meaning |
|---:|---|
| `0` | Control or negative group |
| `1` | Case or positive group |

This workflow is designed for binary comparison.

---

### Step 3: Extract token parameters

```python
params = [
    dataset.samples[i]
    for i in sele_sample
]
```

Each element in `params` corresponds to one selected sample.

Each sample contains MassLinker token parameters.

The three parameter rows are:

| Row | Parameter |
|---|---|
| `0` | weights |
| `1` | centers |
| `2` | sigmas / widths |

These parameters are used by `metabo_dis()` to reconstruct RBF curves and calculate token-level distances.

---

## Distance calculation and caching

The following line calculates JS divergence and Wasserstein distance:

```python
JS, was = metabo_dis(params)
```

This step compares each selected sample with the reference sample used inside `metabo_dis()` and computes metabolite-level distance features.

Because this step can be slow, the results are usually saved:

```python
joblib.dump(JS, filename="cache/JS_distance.joblib")
joblib.dump(was, filename="cache/Wasserstein_distance.joblib")
```

After that, future runs can skip recalculation and load cached results:

```python
JS = joblib.load("cache/JS_distance.joblib")
was = joblib.load("cache/Wasserstein_distance.joblib")
```

Therefore:

- `data/processed_dataset.joblib` is the processed MassLinker dataset;
- `cache/JS_distance.joblib` is an optional cached JS result;
- `cache/Wasserstein_distance.joblib` is an optional cached Wasserstein result.

Only the dataset object is the essential input.

---

## Statistical comparison

The cached or newly calculated distance result is converted to an array:

```python
was_arr = np.array(was)[0]
JS_arr = np.array(JS)[0]
```

Then statistical comparison is performed:

```python
p_values, was_diff = get_diff(
    was_arr,
    group
)
```

The function `get_diff()` compares two groups feature by feature using Welch's t-test.

It returns:

| Output | Description |
|---|---|
| `p_values` | Feature-level p-values |
| `was_diff` | Mean Wasserstein distance difference between the two groups |

The p-values are used to rank differential metabolite tokens.

The `was_diff` values are used in KEGG enrichment scoring.

## Differential feature ranking

```python
plot_p_value(
    p_values,
    met_name,
    mzs=mzs,
    n=20,
    save_path="results/p_value_rank.pdf"
)
```

This plots the top differential m/z features ranked by p-value.

The output file is:

```text
results/p_value_rank.pdf
```

When `mzs` is provided, the function groups features by m/z and keeps the most significant one for each unique m/z.

## Differential peak visualization

```python
plot_peak_comp(
    p_values,
    group,
    params,
    met_name,
    mzs,
    save_path="results/differential_peaks",
    top_n=2000
)
```

This function reconstructs RBF curves for the top-ranked differential features and saves one PDF plot for each selected feature.

Each plot shows:

- reconstructed RBF curves;
- group coloring;
- metabolite name;
- p-value;
- m/z value.

This helps visually inspect whether statistically significant tokens show interpretable peak-level differences.

Note that `plot_peak_comp()` may generate many PDF files when `top_n` is large.

## KEGG enrichment analysis

```python
pathway_df_full = pd.read_csv(
    "metadata/pathway_compound_detail.csv",
    index_col=0
)

kegg_enriched = kegg_enrichment(
    pathway_df_full,
    p_values,
    was_diff
)
```

The function `kegg_enrichment()` maps significant compound-level features to KEGG pathways and performs enrichment analysis.

The returned table includes:

| Column | Description |
|---|---|
| `Pathway_ID` | KEGG pathway ID |
| `Pathway_Name` | KEGG pathway name |
| `P_Value` | Enrichment p-value |
| `Enrichment_Ratio` | Enrichment strength |
| `Hits_in_Pathway (k)` | Number of significant compounds in the pathway |
| `Total_in_Pathway (K)` | Number of compounds in the pathway |
| `Hit_Compounds` | Significant compounds found in the pathway |
| `PPS` | Pathway perturbation-related score |
| `Q_Value (FDR_BH)` | FDR-adjusted q-value |

In the current implementation, `kegg_enrichment()` automatically saves the enrichment result as:

```text
metabolite_enrichment_analysis_results.xlsx
```

This file is saved to the current working directory.

---

## KEGG enrichment visualization

```python
plot_enrichment(
    kegg_enriched.copy(),
    save_dir="results",
    save_name="KEGG_enrichment.pdf"

```

This generates a 3D KEGG enrichment plot.

The plot summarizes enriched pathways using:

| Visual element | Meaning |
|---|---|
| x-axis | Enrichment ratio |
| y-axis | PPS |
| z-axis | Negative log10 p-value |
| color | Enrichment p-value |
| labels | Top pathways ranked by p-value and PPS |

The output file is:

```text
results/KEGG_enrichment.pdf
```

## Significant-feature 2D visualization

The following code selects features with:

```text
p_value <= 0.01
```

```python
was_selected = [
    []
    for i in range(len(was[0]))
]

JS_selected = [
    []
    for i in range(len(JS[0]))
]

for i in range(len(p_values)):
    if p_values[i] <= 0.01:
        for j in range(len(was[0])):
            was_selected[j].append(was[0][j][i])
            JS_selected[j].append(JS[0][j][i])
```

Then PCA and t-SNE visualizations are generated:

```python
plot_2d(
    [JS_selected],
    [was_selected],
    group,
    save_path="results/distance_visualization"
)
```

Output files include:

```text
results/distance_visualization/pca.pdf
results/distance_visualization/tsne.pdf
```

These plots help evaluate whether significant token-distance features separate the two groups.

## Notes

1. The essential input is one `ExcelDataset`-like dataset object.
2. The dataset object should contain `dataset.samples` and `dataset.is_positive`.
3. Cached JS and Wasserstein distance files are optional intermediate results, not mandatory inputs.
4. If cached distance files do not exist, run `metabo_dis(params)` first.
5. The workflow assumes binary group comparison.
6. In this example, group labels are extracted from `dataset.is_positive`.
7. The row order of `pathway_compound_detail.csv` should match the token order in `dataset.samples`.
8. `plot_peak_comp()` can generate many PDF files when `top_n` is large.
9. The example uses `p_value <= 0.01` to select features for 2D visualization.
10. In the current implementation, `kegg_enrichment()` saves `metabolite_enrichment_analysis_results.xlsx` to the current working directory.
11. This workflow is intended for exploratory statistical comparison rather than supervised model training.
12. For larger datasets, model-based classification and validation workflows can be used in separate tutorials.
